# `data/`

Datasets and evaluator weights. Tool *binaries* that used to live here now
live in `src/bin/` (`benchmark`, `selfplay`, `eval-pair-features`,
`relabel-root-value`); see `tools/README.md` and `tools/eval_tuning/README.md`
for the workflows that produce and consume everything below.

```
data/
  weights/                 evaluator weight files (see "Weights")  [tracked]
  datasets/
    battle-factory/        older suite; baked into the crate + tests
    gen9ou/                newer gen9 OU unseen-matchup set        [gitignored]
  suite-shards/            local self-play scratch                 [gitignored]
```

Only load-bearing / non-regenerable files are committed: the crate-baked
battle-factory states, the seeded `matchups-2000.txt` suite, and the weight
snapshots. Everything derivable from the tools is gitignored and rebuilt
locally — see the per-directory notes below.

## Weights

Named `eval-<label>-<feature-count>.weights`. Higher feature counts are newer
schemas. All are plain text parsed by `parse_eval_weights` in
`src/genx/evaluate.rs`.

| File | What it is |
| --- | --- |
| `eval-old-30.weights` | Legacy 30-feature baseline. `run_suite.py`'s default B-side comparison eval. |
| `eval-handcrafted-36.weights` | Historical hand-crafted evaluator. Baked into `evaluate.rs` (`include_str!`) as the regression-gate baseline; reproduce old behavior with `--eval-clamp true`. |
| `eval-adopted-pre-38.weights` | The adopted linear evaluator, before the two experimental 38-schema features were appended. |
| `eval-bench-wish-38.weights` | Adopted evaluator + `BENCH_HP`/`WISH_RECOVERY` candidate weights. Offline-positive but **not** Elo-gated; production keeps these two at zero. |

The current production weights are `DEFAULT_EVAL_WEIGHTS` in `evaluate.rs`, not
a file here. See `docs/CONTEXT.md` for the adopted-evaluator story.

## `datasets/battle-factory/`

The original Gen 9 Battle-Factory-derived suite (no Ubers). Still referenced by
the crate and tests, so treat as load-bearing.

| File | What it is |
| --- | --- |
| `no-ubers-states.txt` | Turn-one battle states. Baked into `evaluate.rs` and read by `tests/test_mcts_tree_reuse.rs`; also the default input to `make_matchup_suite.py`. |
| `no-ubers-dex.json` | Dex snapshot (base stats/types/PP) matching those states. **Gitignored** — only needed to regenerate the states from packed teams. |
| `matchups-2000.txt` | ~2000 seeded, duplicate-free sampled pairings built from the states above by `make_matchup_suite.py`. |

## `datasets/gen9ou/`

Newer Gen 9 OU set: the 504-game unseen-matchup evaluation data. **Gitignored**
(local-only, regenerated by the tools). Built as a pipeline, in order — note
`sample-teams.txt` is the hand-authored source the rest derives from, so keep a
copy if you care about it:

1. `sample-teams.txt` — raw Showdown Import/Export teams (human-readable).
2. `sample-teams-packed.txt` — same teams in packed form
   (`tools/showdown_import_to_packed.js`).
3. `sample-dex.json` — dex snapshot covering those sets.
4. `round-robin-states.txt` — every unordered pair as a turn-one state
   (`convert_battle_factory_teams.py --pairing round-robin`), fed to `selfplay`.

## `suite-shards/`

Sharded self-play inputs, logs, and trajectory dumps (`traj*.jsonl`). Fully
gitignored (`/data/suite-shards/`) — local scratch, safe to delete and
regenerate via `tools/run_suite.py`.
